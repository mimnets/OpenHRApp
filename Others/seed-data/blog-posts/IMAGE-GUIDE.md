# Blog Image Guide — Types, Placements & AI Prompts

This guide covers every blog post in `seed-data/blog-posts/`. For each post, you'll find:
- **What images to add** (cover image + inline images)
- **Where to place them** in the article
- **AI image generation prompts** you can use with Midjourney, DALL·E, Stable Diffusion, or any AI image tool

---

## General Image Guidelines

### Cover Image (Required for every post)
- **Size:** 1200×630px (optimal for social sharing / Open Graph)
- **Aspect ratio:** 1.91:1 (landscape)
- **Format:** WebP (the app auto-converts)
- **Style:** Clean, modern, professional. Use the OpenHR brand colors (primary blue `#4a6fa5`, accent amber `#f59e0b`, accent green `#10b981`) as accent tones.

### Inline Images (1-3 per post)
- **Size:** 800–1200px wide
- **Aspect ratio:** 16:9 or 4:3
- **Format:** WebP
- **Placement:** After the introduction, between major sections, or before the conclusion
- **Purpose:** Break up text, illustrate concepts, show UI or diagrams

### Image Style Consistency
All images across the blog should share a consistent visual language:
- **Modern, clean, minimal** — No clutter, plenty of whitespace
- **Color palette** — Blues, teals, warm neutrals. Avoid oversaturated colors.
- **Illustration style** — Flat/soft vector illustrations for abstract concepts; clean screenshots for UI
- **People** — Diverse, professional, natural (not stiff stock photo poses)
- **No cheesy stock photography** — Avoid the "person staring at laptop smiling" cliché

## AI Image Generation — Quick Tips

### For Midjourney
Add these parameters to the prompts above:
- `--ar 16:9` for inline images, `--ar 1.91:1` for cover images
- `--style raw` for more photorealistic results
- `--v 6.1` for the latest model
- Add `--no text, --no words, --no letters` to prevent AI from generating garbled text

Example:
```
A clean modern flat illustration of a small business owner... --ar 1.91:1 --v 6.1 --no text words letters
```

### For DALL·E (via ChatGPT)
- Use the prompts as-is — DALL·E handles natural language well
- Request "wide" format for covers (1792×1024) and "standard" for inline (1024×1024)
- Add: "Do not include any text, words, or letters in the image."

### For Stable Diffusion
- Use the prompts with a model like SDXL or Stable Diffusion 3
- Add negative prompts: `text, watermark, signature, low quality, blurry, distorted faces, extra fingers`
- Aspect ratio: `--ar 16:9` or set width/height directly

### General Tips
1. **Generate 3-4 variations per prompt** and pick the best one
2. **Check for AI artifacts** — distorted hands, weird faces, garbled text
3. **Upscale** the chosen image before using
4. **Convert to WebP** before uploading (OpenHR auto-converts, but pre-converting saves storage)
5. **Keep file sizes under 200KB** for fast loading — use TinyPNG or Squoosh if needed
